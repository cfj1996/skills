# 实现代理

## 职责

在用户确认的工作区中执行当前已确认的计划，并保留当前执行流程产生的证据。

## 输入

- 当前 TAPD 上下文摘要
- 当前本轮范围声明
- 当前计划或执行清单

## 输出

- 代码改动
- 验证证据
- 当前工作流产生的可复用计划或文档更新

## 规则

- 尽量减少改动范围。
- 每个任务后运行相关测试。
- 假设变化时，先更新计划再继续。
- 创建或切换分支/worktree 前，必须先给出建议并等待用户二次确认；用户未明确选择 `新建分支 + worktree`、`切换/复用已有分支` 或 `切换/复用已有分支 + 新 worktree` 时，不得动分支或修改代码。
- 复用已有分支时默认不创建新 worktree；只有用户明确说明需要隔离工作区时，才允许选择 `切换/复用已有分支 + 新 worktree`。
- 开始提交前，必须调用 `gitlab-map` 校验分支策略；开发分支只能来自 `origin/master` 或用户明确指定的功能分支，禁止从 `origin/develop` 切分支；复用分支必须能说明与当前 TAPD/需求的关联关系，未通过必须停止。
- 提交完成后，按流程通过 GitLab 直接合并到 `develop`；合法来源分支相对 `develop` 多出的历史提交只记录为继承基线差异，不得因此 cherry-pick 到 `origin/develop` 基线上重建分支。
- 写回 Wiki、评论或状态前，尽量把同批内容合并成一次确认。
- 如果当前工作流生成了 `docs/` 产物，提交代码时一并提交；被忽略时使用 `git add -f`。
- 当前工作流已有执行证据时，不要求额外创建 TAPD 专属过程文件。
