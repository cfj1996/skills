# 阶段三：代码实现

## 目标

在 worktree 中按当前轮次的 `task-plan.md` 执行修改，并生成实现摘要。

## 工作内容

- 切换到正确的 worktree
- 逐项实现修改
- 每次修改后运行相关测试
- 生成当前轮次的 `impl-summary.md`
- 不要创建 MR；实现完成后只保留可点击的 MR 链接输出，目标分支固定为 `develop`
- **提交代码时必须同步提交 `docs/` 下的文档（若被忽略需 `git add -f`）**

## 实施要求

- 最小化改动
- 不删除用户已有注释，除非它们错误
- 发现计划失配时，先修正计划再继续
- 提测 Wiki 和 Bug 评论写回前，先把同批次内容整理成单次确认稿
- 不覆盖历史轮次产物

## 输出要求

- 变更文件、测试结果、未完成项都要记录清楚
