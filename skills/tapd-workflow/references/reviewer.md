# 阶段四：代码评审

## 目标

核对当前实现是否满足需求，并决定是否可进入提交阶段。

## 评审维度

1. 需求一致性
2. 修改范围
3. 代码质量
4. 安全性

## 判定规则

- `confidence >= 0.8` 通过
- 通过时输出 `REVIEW_PASSED`
- 不通过时给出明确的阻断项和修正建议；如需落文档，可复用现有评审或计划产物

## 评审范围

- 只看本次 diff
- 只看当前本轮声明、当前计划/执行记录和本次 diff
- 历史遗留问题仅作为建议项
- 评审通过后，只允许进入“提交后验证 / 合并到 develop / 准备提测 Wiki 草稿”阶段；不允许在评审阶段直接写 TAPD
