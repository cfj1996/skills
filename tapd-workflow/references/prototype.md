# 原型处理规范

## 识别范围

仅扫描以下范围：

- TAPD 详情页正文
- 评论正文
- 详情页和评论中的图片/截图

## 识别目标

- `lanhuapp.com`
- `mastergo.com`
- `figma.com`
- 其他可访问原型站点

## 处理步骤

1. 识别详情页和评论中的候选链接。
2. 使用 `chrome-devtools-mcp` 打开主原型链接。
3. 读取原型中默认展示的需求文档或说明区域，不主动切换到其他需求文档。
4. 判断页面结构、流程状态、关键交互、显式限制和验收线索。
5. 从图片/截图中补充识别可见链接、页面状态和设计意图。
6. 如有多页面或多状态，记录主路径和补充路径；如存在多个需求文档，仅记录默认展示文档的结论，并注明其他文档未读取。
7. 将提炼结果写入 `item-context.md` 的原型分析部分，明确区分“原型文档”与“提炼后的需求文档”。

## 输出内容

- 原型链接
- 原型文档中的默认需求文档摘要
- 设计意图
- 关键交互点
- 约束条件与未明确项
- UI 截图分析
- 无法访问时的原因
